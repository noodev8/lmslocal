**Error Tracking Explained**:
Error tracking tools (like Sentry or Firebase Crashlytics) automatically:
- Capture app crashes and send crash reports to a dashboard
- Track non-fatal errors (API failures, exceptions)
- Show which device/OS version caused crashes
- Provide stack traces for debugging
- Alert you when new errors occur

**Without error tracking**: You won't know if the app crashes for users unless they tell you.

  - **Option 2**: Use Sentry (more powerful, self-hosted or cloud)
  - **Option 3**: None (rely on user reports and server logs)
  - **Recommendation**: At minimum, use Crashlytics (free and catches production crashes you won't see in testing)
